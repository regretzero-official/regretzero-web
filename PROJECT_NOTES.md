# RegretZero Project Notes

## 1. Product identity

RegretZero is not a generic finance content site.
It should feel like a premium, simple investment experience tool.

Core promise:
- See what would have happened if you invested 10 years ago
- Compare with a benchmark
- Use that experience to make calmer decisions today

The service should not feel like:
- a phishing-style landing page
- an AI-generated template site
- a noisy dashboard full of disconnected widgets

The service should feel like:
- simple
- trustworthy
- sharp
- premium
- easy for beginners

## 2. Tone rules

Use Korean that sounds like a real person wrote it.

Avoid:
- translation-style copy
- abstract slogans
- overexplaining
- fake-emotional copy
- too many labels and helper sentences

Preferred copy style:
- short
- concrete
- action-oriented
- human

Good direction:
- "그때 샀으면 지금 어땠을까?"
- "마음에 드는 자산 하나만 골라보세요."
- "10년 결과를 바로 볼 수 있어요."

Bad direction:
- "실제 데이터 기반으로 다시 계산합니다"
- "후회를 길게 설명하지 않습니다"
- "한 번 보면 바로 감 옵니다"

## 3. Design rules

Main rule:
- one message per screen

Visual direction:
- bright and simple
- not dark, scammy, or overly dramatic
- clean left alignment
- strong spacing consistency
- minimal text

Do:
- use one clear headline
- keep supporting text to one short line when possible
- align logo, headline, hero image, example card, and CTA on the same left axis
- use cards sparingly
- keep CTA hierarchy obvious

Do not:
- stack multiple explanation paragraphs at the top
- mix too many tones on one screen
- use too many decorative sections above the fold

## 4. Landing page principles

The landing page is currently the biggest quality risk.

The landing must be rebuilt as:
- logo/header
- short headline
- core hero image
- one example card
- investment input
- main CTA

Webtoon/comic should support understanding, but should not create clutter.

Above the fold should answer only:
- what is this
- why is it interesting
- what do I click now

## 5. Investment features principles

The current problem is not missing features.
The current problem is fragmented product identity.

`버티기 코치` and `미래 전략` should become one integrated decision flow.

Target structure:
- 지금 내 상태
- 오늘의 행동
- 앞으로 90일
- 10년 계획

This should become a single product concept, for example:
- `내 투자 플랜`
- `10년 투자 콘솔`

## 6. Trust rules

Trust is broken by:
- broken Korean text
- mojibake
- too many AI-like helper cards
- unclear labels
- fake prediction wording

Trust is improved by:
- clean Korean
- fewer sections
- clear calculation logic
- clear distinction between past data and future scenarios
- one coherent flow

## 7. Immediate priorities

Priority 1:
- fix and simplify landing

Priority 2:
- rebuild HoldCoachPanel into a clean decision tool

Priority 3:
- bring future strategy back as part of one integrated investment plan flow

Priority 4:
- remove or demote noisy sections that look like demo widgets

## 8. Non-negotiables

- No broken Korean strings
- No phishing-site tone
- No clutter-first layout
- No adding more features before the core flow is coherent
- Keep engines if they are working; rebuild presentation if needed
