# Regretzero 외부 검수 요청 패킷

아래 문서를 Gemini에 그대로 붙여넣어 Regretzero의 제품성, UI/UX, 카피, 계산 신뢰도, 법적 안전성, 모바일/PC 완성도, 재방문/공유 가능성을 다시 크로스체크하세요.

---

## 0. Gemini에게 보낼 역할 지시

너는 지금부터 Regretzero 웹서비스의 외부 제품 감사팀 역할을 맡아줘.

동시에 다음 역할을 수행해줘.

- 핀테크 UX 리드
- 토스/토스증권식 모바일 UI 디자이너
- 투자 초보자 교육 기획자
- 금융 서비스 카피라이터
- 프론트엔드 QA 엔지니어
- 데이터 계산 신뢰도 검증자
- 금융 컴플라이언스 리뷰어
- 사용자 리서처
- 제품 PM
- 바이럴/공유 UX 그로스 담당자

목표는 Regretzero를 “AI로 대충 만든 투자 계산기”가 아니라, 실제 사용자가 저장하고 공유하고 다시 들어오고 싶은 **10년 전 투자 시뮬레이터**로 완성시키는 것이다.

검수 대상 URL:

- 배포 사이트: `https://www.regretzero.kr`
- 로컬 개발 서버를 볼 수 있다면: `http://localhost:3000`

검수 시 전제:

- Regretzero는 투자 추천 서비스가 아니다.
- 특정 종목의 매수·매도를 권유하면 안 된다.
- 과거 데이터를 바탕으로 “10년 전에 샀다면 얼마가 되었는지”와 “그 결과까지 어떤 하락, 회복, 기다림을 지나야 했는지”를 보여주는 체험형 서비스다.
- 사용자는 ETF, QQQ, CAGR, MDD 같은 용어를 모를 수 있는 투자 초보자까지 포함한다.
- 서비스는 토스/토스증권처럼 쉽고, 믿을 수 있고, 모바일에서 부드러워야 한다.
- 과장된 AI 랜딩페이지, 번역기 말투, 의미 없는 장식, 방어적인 금융문서 톤은 피해야 한다.

## 1. 서비스 정체성

Regretzero의 핵심 문장:

> 10년 전 샀다면, 지금 얼마였을까요?

사용자가 체험해야 하는 질문:

- 10년 전에 이 자산을 샀다면 지금 얼마가 되었을까?
- 같은 돈을 다른 자산에 넣었다면 결과가 얼마나 달라졌을까?
- 최종 금액만 보면 쉬워 보이지만, 중간에는 얼마나 크게 흔들렸을까?
- 나는 그 하락과 회복 기간을 실제로 버틸 수 있었을까?
- 놓친 10년보다 앞으로의 10년을 어떻게 준비해야 할까?

원하는 감정:

- 처음에는 “아, 그때 샀으면 어땠을까?”라는 호기심
- 레이스 중에는 “이렇게 흔들렸다고?”라는 몰입감
- 결과에서는 “수익률만 보고는 몰랐던 시간이 있었구나”라는 이해
- 마지막에는 “다음 10년에는 내 기준을 가져야겠다”는 결론

검수해줘:

- 첫 화면 3초 안에 서비스가 무엇인지 이해되는가?
- “10년 전 투자 시뮬레이터”라는 정체성이 분명한가?
- 후회를 자극하되 공포 마케팅이나 투자 권유처럼 보이지 않는가?
- 사용자의 첫 행동이 바로 보이는가?
- 초보자에게 금융 앱처럼 어렵거나 무섭게 느껴지지 않는가?

## 2. 현재 구현된 주요 기능

### A. 홈 화면

현재 홈 화면은 대표 예시 중심이다.

주요 요소:

- 헤드라인: `10년 전 샀다면, 지금 얼마였을까요?`
- 설명: `그때 샀다면 얼마가 됐는지, 그리고 그 사이에 얼마나 흔들렸는지 함께 봅니다.`
- 투자권유 오인 방지 고지: `투자 권유가 아닌 과거 시뮬레이션입니다. 과거 기록은 미래 수익을 보장하지 않으며, 실제 투자 판단은 본인 책임입니다.`
- 대표 예시 카드:
  - `10년 전 엔비디아를 샀다면?`
  - `10년 전 비트코인을 샀다면?`
  - `10년 전 테슬라를 샀다면?`
  - `엔비디아 vs SK하이닉스`
  - `테슬라 vs 현대차`
  - `SOXL vs QQQ`
- 보조 행동:
  - `더 많은 예시 보기`
  - `한 종목만 보기`
  - `직접 고르기`
  - `처음이라면 30초만 보고 시작하세요`

검수해줘:

- 대표 예시가 호기심을 만들지만 투자 추천처럼 보이지 않는가?
- 엔비디아, 테슬라, 비트코인 등 급등 자산이 전면에 있을 때 법적·윤리적 리스크가 커 보이지 않는가?
- 예시 카드의 제목과 설명이 자연스럽고 고급스럽게 느껴지는가?
- 초보자가 “무엇을 눌러야 하는지” 바로 알 수 있는가?
- 비기너 가이드가 너무 숨어 있거나 반대로 너무 주절주절 보이지 않는가?
- 모바일 첫 화면에서 핵심 행동이 가려지거나 하단 메뉴와 충돌하지 않는가?

### B. 두 자산 비교 흐름

사용 흐름:

1. 대표 예시 또는 직접 선택으로 자산 2개를 고른다.
2. 금액을 정한다.
3. 투자 방식 선택:
   - 한 번에 투자
   - 매달 적립
4. 차트 레이스를 본다.
5. 결과 화면에서 수익, 낙폭, 회복, 기다림을 본다.

주요 기능:

- 자산 검색
- 카테고리 필터
- 전체 자산 보기
- 대표 자산 빠른 선택
- 금액 프리셋
- 만원 단위 직접 입력
- 적립식 월 납입액 입력
- 레이스 배속: `0.5x`, `1x`, `2x`, `4x`
- 현재 1위, 현재 격차, 기준값 표시
- 주요 순간 마커
- 결과 공유
- 결과 저장

검수해줘:

- 선택 → 금액 → 레이스 → 결과 흐름이 끊기지 않는가?
- 자산 카드가 초보자에게 이해되는 이름과 설명을 갖고 있는가?
- 영문 티커(QQQ, SOXL 등)가 너무 앞에 나와 어렵게 느껴지지 않는가?
- 금액 입력이 만원 단위로 자연스러운가?
- 적립식과 일시 투자 차이가 명확한가?
- CTA 문구가 흐름마다 일관적인가?
- 모바일에서 스크롤, 하단 고정 버튼, 하단 메뉴가 충돌하지 않는가?

### C. 한 종목만 보기

최근 한 종목 보기 흐름을 두 자산 비교와 통일했다.

현재 의도:

- 한 종목을 선택하면 `선택 종목 vs 원금 기준선` 형태로 레이스한다.
- 두 자산 비교처럼 차트 레이스, 배속 조절, 결과 분석표를 제공한다.
- 한 종목만 보고 끝나는 것이 아니라 `다른 자산과 비교`로 이어질 수 있어야 한다.

검수해줘:

- 한 종목 보기와 두 자산 비교의 UI 통일성이 충분한가?
- `원금`을 비교 대상으로 넣는 방식이 직관적인가?
- 사용자가 “왜 한 종목인데 레이스를 하지?”라고 헷갈리지 않는가?
- 한 종목 결과 화면이 두 자산 결과 화면만큼 상세하고 신뢰감 있는가?
- 한 종목 보기의 위치가 메인 기능을 흐리지 않는가?

---

### D. 차트 레이스

Regretzero의 핵심 경험이다.

주요 기능:

- 선택 자산의 평가금액 흐름을 애니메이션으로 보여준다.
- 순위가 바뀌면 카드 순위도 바뀐다.
- 한 종목 보기에서는 `원금` 기준선과 함께 레이스한다.
- 배속 조절을 지원한다.
- 다시보기, 재생/일시정지를 지원한다.
- 주요 순간 마커를 표시한다.
- 레이스 종료 후 결과로 이어진다.

검수해줘:

- 레이스가 이 서비스의 킬러 기능처럼 느껴지는가?
- 차트와 순위 카드가 한 화면에서 자연스럽게 보이는가?
- 모바일에서 차트를 보려고 스크롤을 많이 해야 하지 않는가?
- 배속 버튼 위치가 엄지손가락 동선에 맞는가?
- 결과 팝업이나 CTA가 차트를 가리지 않는가?
- 누가 이겼는지보다 “언제부터 갈라졌는지”가 느껴지는가?
- 차트 문구가 고리타분하거나 설명 과잉으로 보이지 않는가?

---

### E. 결과 화면

결과 화면은 단순 숫자 나열이 아니라 “10년 투자 타임머신”이어야 한다.

현재 표시 정보:

- 한 줄 결론
- 최종 승자
- 최종 금액
- 총 수익률
- 최종 배수
- CAGR 또는 적립식 누적 수익률/XIRR 관련 표시
- 실제 적용 시작일
- 종료 날짜
- 시작 가격
- 종료 가격
- 투입 금액
- 최종 평가액
- 수익
- 최대 낙폭
- 고비 횟수
- 회복 기간
- 버티기 난이도
- 가장 힘들었던 구간
- 주요 고비
- 10년 기다림 지도
- 결과 공유
- 결과 저장

현재 추가된 초보자 도움말:

- `CAGR`
- `최대 낙폭`
- `회복 기간`
- `고비 횟수`

검수해줘:

- 결과 화면이 너무 길거나 피곤하지 않은가?
- 가장 중요한 결론이 먼저 보이는가?
- 초보자가 `최종 금액`, `최대 낙폭`, `회복 기간`의 의미를 이해할 수 있는가?
- 결과 문구가 결과 조건에 따라 자연스럽게 달라지는가?
- 수익이 큰 자산을 보여줄 때 투자 권유처럼 느껴지지 않는가?
- 적립식 결과에서 CAGR과 누적 수익률이 혼동되지 않는가?
- 저장/공유 버튼의 위치와 문구가 자연스러운가?

### F. 버티기 / 10년 기다림 지도

현재 방향:

- `버티기`는 독립 탭에서 내리고, 결과 화면의 핵심 분석 섹션으로 통합했다.
- 최종 수익률보다 “그 결과까지 버텨야 했던 시간”을 보여준다.

주요 구성:

- 버티기 난이도
- 최대 낙폭
- 회복 기간
- 신고가 없이 기다린 최장 기간
- 10년 기다림 지도
- 월별 상태:
  - 새 고점
  - 기다림
  - 깊은 대기
  - 원금 아래
- 가장 힘든 구간 카드
- 상세 고비 차트

검수해줘:

- `10년 기다림 지도`가 처음 보는 사람에게 직관적인가?
- 색상 의미가 바로 이해되는가?
- “무수익 시간표”보다 “기다림 지도”라는 이름이 더 좋은가?
- 이 기능이 단순 부가 기능이 아니라 Regretzero의 차별화 요소로 느껴지는가?
- 결과 화면 안에서 위치가 적절한가?
- 지도와 주요 고비 카드가 너무 정보 과밀로 보이지 않는가?

### G. 메뉴 / 내비게이션

최근 메뉴 구조를 크게 정리했다.

현재 모바일 하단 메뉴:

- `예시`
- `비교`
- `기록`
- `더보기`

현재 PC 상단 메뉴:

- `대표 예시`
- `직접 고르기`
- `기록`
- `더보기`

현재 더보기 메뉴:

- `사용 가이드`
- `소개`
- `제휴·문의`

최근 제거/숨김 처리한 더보기 메뉴 항목:

- `대표 예시 보기`
- `한 종목만 보기`
- `직접 고르기`
- `저장한 기록`
- `시작 시점 바꾸기`
- `금액 계산기`
- 독립 `버티기` 메뉴

의도:

- 더보기는 “도움말과 문의”만 남긴다.
- 실제 비교 행동은 하단 메뉴/상단 메뉴에서만 시작한다.
- 메뉴 중복을 줄여 초보자가 길을 잃지 않게 한다.

검수해줘:

- 하단 메뉴 4개가 적절한가?
- `예시`, `비교`, `기록`, `더보기`라는 이름이 초보자에게 명확한가?
- 더보기에 지원 메뉴만 남긴 방식이 좋은가?
- `홈` 버튼을 하단에서 제거한 결정이 적절한가? 상단 `Regretzero` 로고 홈 이동만으로 충분한가?
- `버티기`를 독립 메뉴에서 내리고 결과 화면으로 통합한 것이 맞는가?
- `/calculator` 라우트는 남아 있지만 메뉴에서는 숨긴 상태다. 완전 삭제하는 것이 좋은가, 숨겨두는 것이 좋은가?

## 3. 계산 / 데이터 신뢰도 검수 항목

현재 구현 방향:

- 일시 투자:
  - 시작일 가격 기준 매수 수량 계산
  - 현재/마지막 가격 기준 최종 평가액 계산
  - 총수익률, CAGR, 최대 낙폭, 회복 기간, 고비 구간 계산
- 적립식 투자:
  - 매월 납입액으로 누적 매수
  - 총 납입 원금, 최종 평가액, 수익률 계산
  - CAGR 혼동을 피하고 누적 납입 원금 대비 수익률 중심으로 표시
- 예금:
  - 비교 기준으로 연 2.7% 복리 가정
- 부동산:
  - 실제 개별 매물 가격이 아니라 지수/프록시 흐름
- 해외 자산:
  - 원화 환산
- 10년 데이터가 없는 자산:
  - 가능한 가장 이른 날짜부터 계산하고 사용자에게 안내

검수해줘:

- 일시 투자 계산 설명이 충분한가?
- 적립식 수익률이 오해 없이 표시되는가?
- 예금 2.7% 고정 가정이 보수적/비현실적으로 보이지 않는가?
- 부동산 프록시 데이터 한계를 충분히 설명하고 있는가?
- 원화 환산 기준이 충분히 신뢰감 있게 설명되는가?
- 10년 미만 데이터 자산 안내가 충분히 눈에 띄는가?
- 결과 화면 하단 데이터 주석이 너무 숨겨져 있지 않은가?

## 4. 최근 구현 / 수정 내역

최근 반영된 개선:

- 한글 깨짐이 있던 메뉴/문서 파일을 UTF-8 기준으로 재작성
- 홈/결과/한 종목 결과에 투자 권유 오인 방지 고지 추가
- `CAGR`, `최대 낙폭`, `회복 기간`, `고비 횟수` 지표에 도움말 팝오버 추가
- 모바일 하단 메뉴가 보이는 화면의 하단 안전 여백 보강
- 자산/금액 단계처럼 고정 CTA가 있는 화면의 모바일 하단 여백 보강
- 한 종목 보기를 `선택 종목 vs 원금 기준선` 레이스로 통합
- 하단/상단 주요 메뉴에서 독립 `버티기` 제거
- 더보기 메뉴를 `사용 가이드`, `소개`, `제휴·문의`만 남기는 지원 메뉴로 정리
- 더보기 안의 중복 기능 제거:
  - 대표 예시 보기
  - 한 종목만 보기
  - 직접 고르기
  - 저장한 기록
  - 시작 시점 바꾸기
  - 금액 계산기

최근 검증 결과:

- `npm run lint` 통과
  - 기존 경고 1개 유지: `components/TimeMachineSimulator.tsx`의 `<img>` 사용 경고
- `npm run test:unit` 통과
  - 30 files / 112 tests
- `npm run build` 통과
- 깨진 한글 스캔 결과:
  - 실제 활성 소스에서는 문제 없음
  - 테스트 파일의 mojibake 탐지용 정규식만 검출됨
- 브라우저 메뉴 확인:
  - 더보기 패널 안에는 `사용 가이드`, `소개`, `제휴·문의`만 노출
  - `금액 계산기`, `시작 시점`, `한 종목만 보기`, `저장한 기록`은 더보기에서 제거됨

검수해줘:

- 최근 수정 방향이 맞는가?
- 메뉴를 너무 줄여서 발견성이 떨어지지는 않는가?
- 더보기에 `제휴·문의`를 둔 것이 자연스러운가?
- 아직 남은 메뉴/CTA 중 중복 또는 혼란이 있는가?

## 5. 알려진 남은 리스크

현재 남은 리스크:

- PC 결과 화면은 아직 전용 2열 레이아웃으로 충분히 고급스럽게 정리되지 않았다.
- 공유 UX는 아직 텍스트/링크 중심이며, 네이티브 공유 카드나 카카오톡 공유 템플릿은 부족하다.
- `/calculator` 라우트는 메뉴에서 숨겼지만 아직 빌드에 남아 있다.
- 급등 자산 예시가 많아 투자 권유처럼 보이지 않게 계속 조심해야 한다.
- 데이터 출처/한계 설명은 더 잘 보이게 다듬을 여지가 있다.
- 결과 화면은 기능이 많이 누적되어 길고 피곤하게 느껴질 수 있다.
- 가이드/소개 페이지의 문체가 전체 제품 톤과 완전히 맞는지 추가 검수가 필요하다.

검수해줘:

- 이 리스크 목록이 정확한가?
- 더 큰 리스크가 빠져 있는가?
- 당장 고쳐야 할 24시간 우선순위와 1주일 우선순위를 나눠줘.

문구가 AI스럽거나 번역기처럼 느껴지는 곳을 찾아서 교체안을 줘.

특히 아래 위치를 봐줘.

- 홈 메인 헤드라인

- 대표 예시 카드 제목/설명
- 자산 선택 화면
- 금액 선택 화면
- 레이스 화면
- 결과 화면 한 줄 결론
- 최대 낙폭/회복 기간/버티기 난이도 설명
- 기다림 지도 문구
- 사용 가이드
- 소개 페이지
- 디스클레이머
- 더보기 메뉴 문구

각 항목마다:

- 현재 문구의 문제
- 추천 문구 3개
- 가장 추천하는 최종안 1개

가상의 사용자 60명을 아래처럼 나눠 테스트했다고 가정하고 결과를 줘.

- 투자 완전 초보 15명
- 월급 적립식 관심자 10명
- 주식 경험자 10명
- 코인/테크 관심자 5명
- 부동산 관심자 5명
- 모바일 사용성이 낮은 사용자 5명
- 금융/증권업 실무자 5명
- 프론트엔드 QA 5명

각 그룹별로:

- 처음 이해한 내용
- 가장 헷갈린 지점
- 이탈 가능성이 높은 순간
- 마음에 든 핵심 포인트
- 공유/재방문 의사
- 반드시 고쳐야 할 점

### 7. 법적/윤리적 안전성 점검
