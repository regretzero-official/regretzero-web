# Regretzero Product Goal

## Product Identity

Regretzero is an investment time-machine service, not a plain return calculator.

It helps users answer:

- What if I had bought this asset 10 years ago?
- What if I had invested a lump sum?
- What if I had invested every month?
- Could I really have survived the drawdowns and boring sideways years?
- What should I prepare for in the next 10 years?

The product begins with regret, but should end with preparation.

## Core Promise

Show how the same money lived through different time paths.

The service must make long-term investing feel visible:

- Final value
- Total return
- Annualized return for lump-sum investing
- XIRR or clearly labeled cumulative return for monthly investing
- Maximum drawdown
- Recovery period
- Hard periods and survival difficulty
- Shareable, repeatable scenarios

## Product Principles

- Mobile first.
- Clear enough to understand within 3 seconds.
- Friendly enough for beginners.
- Trustworthy enough that calculation labels do not mislead.
- Emotional, but not fearmongering.
- Never imply investment advice, guaranteed returns, or future certainty.
- Reuse the existing code, data, routing, and calculation structure whenever possible.

## MVP Evaluation Criteria

Each `/goal` loop scores the product across 10 areas:

1. Core value clarity
2. Basic simulation completeness
3. Calculation trust
4. Result screen immersion
5. Race chart education
6. Save/share experience
7. Mobile UX
8. Frontend quality
9. Performance and stability
10. Legal and ethical safety

Score guide:

- 90+: MVP product quality. Polish remaining details.
- 75-89: Usable, but UX/calculation/share quality still needs work.
- 60-74: Core function exists, but still feels like a calculator.
- Below 60: Rework the core flow before adding features.

## Current Product Direction

The highest priority is not adding random features. It is making the existing flow feel reliable and worth sharing:

1. Pick assets.
2. Choose lump-sum or monthly investing.
3. Watch the race.
4. Understand the result and the pain required to reach it.
5. Save or share the result.
6. Try another asset or start date.
